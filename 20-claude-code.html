ClaudeCode给中国玩了把特洛伊木马

当编程助手把监视藏进看不见的标点符号，信任是如何塌房的

作者 广谱 | 2026年7月14日 | 约 7 分钟
纽约时报评论版风格 · 冷木刻插画
导读
2026年6月底，有安全研究员从 Claude Code 中逆向出一段专门标记中国用户的代码：比对本地时区、扫描147个中国域名，再以肉眼难辨的水印回传。事件引发中美 AI 监管与信任的连锁震荡。广谱说：我们到底把多少权力，交给了桌面上那个输入框？
原声 · ORIGINAL VOICE

2026年6月30日，海外社交平台 Reddit 上，一名技术人员发帖说，Anthropic 公司的 Claude Code 里，藏着一段专门标记中国用户的监视代码。随后安全研究员 Adnane Khan 在 GitHub 发布逆向分析报告，抠出了完整机制，而这一切早在4月2日的 2.1.91 版就埋进去了。

它干的事很隐蔽：读取你电脑的本地时区，重点比对上海和乌鲁木齐，再扫描你的 API 地址，跟 147 个中国域名清单比对——百度、阿里、字节、美团、携程、小红书、京东、B站、月之暗面、MiniMax、阶跃星辰都在。一旦命中，它不弹窗、不报错，而是悄悄改写系统提示词里的日期——把连字符换成斜杠，把 Today's 里的撇号换成肉眼难辨的水印，发回服务器。用户全程无感。

7月2日 Anthropic 推送新版，删掉了这段代码；7月8日，工信部网络安全威胁和漏洞信息共享平台发布风险提示，点名 2.1.91 到 2.1.196 版本存在安全后门；阿里更早——7月3日内部通知，7月10日起全员禁用 Claude Code，统一换自研的 Qoder。Anthropic 把这套机制解释为3月起的实验，防账号倒卖和模型蒸馏，并称已发现 2.5 万个阿里关联虚假账号、2800 万次疑似蒸馏调用。

⚡ GLOBAL FOCUS · 回响 ECHOES

Reddit r/ClaudeAI：有网友替 Anthropic 辩护——"你对 Claude 获取你的位置信息感到震惊？你电脑上有上百种东西在监视和追踪你。"另一派则反驳发现者："虽 IP 保护合理，但不应以在每位开发者系统上嵌入实质间谍软件为代价。"也有中间派：把这事定性为间谍软件"过于夸大"，但"永远不要过于夸大"。

知乎：一条高赞评论点出信任模型的崩塌——一个被允许读写文件和执行命令的 agent，同时在悄悄检测你的网络环境并把结果上报。今天它能报告时区和域名，明天就能偷掉你所有的数据。

微博：有人写道：工信部监测发现，它内置了一套暗哨系统，能在用户毫无察觉的情况下窃取地域、身份等关键数据。

广谱说 · SPECTRUM

这场风波最值得琢磨的，不是谁在偷看谁，而是我们究竟把多大的权力交给了桌面上那个输入框。

Claude Code 这类代理式编程工具，默认拥有读文件、跑命令、操作 git 的权限——这已经逼近红线。我们授权它，前提是行为我看得见。可当一款工具把分类逻辑藏进看不见的标点符号，信任就塌了。

虎嗅把它称作中美 AI 监管双向闸口的起点，并提出"硅幕"一词。Anthropic 的服务条款从一开始就排除中国大陆，2025年9月又禁止中国控股超50%的企业使用；6月底到7月初，大量中国账户被无预警封禁，杭州成了重灾区，圈内甚至流传近期去杭州出差别开 Claude。当开发工具本身成为大国博弈的筹码，每一行代码都可能暗含政治意志——这不是阴谋论，是 147 个域名清单和大规模惨烈封号写下的事实。

隐写术本身不新。从古希腊把消息刺在奴隶头皮上，到现代数字水印，都是让第三者看不见、让接收方读得懂的老手艺。Anthropic 把它用进消费级开发工具，手法之精巧反而更刺眼：它本可以用透明的遥测字段、写进变更日志，却选了混淆与隐写。有开发者打了个比方——一家餐厅高调展示卫生评分，后厨却把昨天的鱼重新加热。品牌承诺和厨房现实之间的缝，就是信任塌架的地方。

品牌承诺和厨房现实之间的缝，就是信任塌架的地方。

真正该留下的，是两条朴素原则：最小权限与数据隔离。任何 AI 工具，坚决不授予超出需求的系统权限，别让它直接碰企业核心代码、客户隐私、财务凭证。工具好不好用是其次，底线是——你玩代码时，别让代码反过来玩了你。
